このサイトは、個人ホームページとしての最小限の自己紹介を持たせつつ、主目的を「使ったことのあるスクリプトの記録と公開」に置いています。

見た目の基調は Copper File に寄せ、書類棚や保管庫のように落ち着いて積み上げていける雰囲気を目指しています。

## 現在の公開方針

- 公開先の GitHub アカウントは最後に決める
- それまではローカルで作成と確認を進める
- 公開先依存の設定は別ファイルに分けて管理する

## 現在のカテゴリ方針

- 現時点では `PowerShell` と `いたずら用` の2カテゴリで運用する
- カテゴリの細分化は、記事数が増えてから行う

## デザイン方針

- Copper File を既定テーマとして全体に適用する
- 派手さよりも、保管しやすさと一覧性を優先する
- 長期運用しても古びにくい、硬めの記録庫らしい見た目を保つ

## このサイトで重視すること

- 後から見ても再利用しやすいこと
- スクリプトの目的と実行方法が明確であること
- 公開して問題のない内容だけを載せること
- Markdown ベースで継続更新しやすいこと

## 掲載しないもの

- API キーや認証情報を含むもの
- 業務固有の情報を含むもの
- 再現が難しすぎるもの
- 第三者に誤用されやすいもの

## 次の拡張候補

- タグ別一覧の追加
- 更新履歴ページの追加
- GitHub リポジトリへの導線強化

## 運用メモ

公開先を最後に決める運用の詳細は [docs/publish-later-workflow.md](docs/publish-later-workflow.md) を参照。
